# 抖動演算法實務差異：社群共識整理

> 來源：LightBurn 官方論壇（最豐富）、LA Hobby Guy、Snapmaker 論壇、實作者部落格（Lùmen、awesome.tech、craftgineer、OMTech Norton tile 指南）。2026-07-04 整理，供照片雕刻功能規劃使用。

## 一句話總結

> **演算法是最後一個變數。** 社群反覆糾正的優先序是：對焦（最小光斑）→ DPI 與光斑尺寸匹配 → 功率/速度校準 → 影像前處理 → 最後才是選哪個抖動。「哪個演算法好」在實務上多半是口味問題；「DPI 開太高」才是照片雕壞的頭號原因。

## 演算法之間真的看得出差別嗎？

**看得出，但屬二階差異**，且與「尺寸/DPI」的關係大於「材質」：

- 懷疑派：「它們只是排列點的不同方法」（LightBurn 論壇經典回答）；花崗岩實測「Stucki、Jarvis 幾乎無差」。
- 實測派（白磁磚 Stucki vs Jarvis 對照）：**Stucki 顆粒感較重但細節保留多**（眼睛反光裡的柵欄細節 Jarvis 丟了）；**Jarvis 漸層平滑、第一眼銳利但細節流失**。結論分歧點是口味（有人愛 Stucki 的「Kodak Tri-X 底片感」）。
- 最實用的框架（Lùmen）：**平滑系（Jarvis）適合大尺寸**（200×300mm 驚豔、50×50mm 變糊）；**銳利系（Atkinson/Stucki）小尺寸站得住**。

## 各模式的實際使用場景

| 模式 | 實務上何時選用 |
|---|---|
| **Atkinson** | **社群預設起手式**——木頭/皮革上的人像與 logo；高對比、寬容度高。注意極亮/極暗區會爆掉 |
| **Jarvis** | 大尺寸、漸層多（風景、膚色）；最平滑但最軟，小圖會糊 |
| **Stucki / Floyd-Steinberg** | 中間值：細節保留型。兩者互相冗餘——**擇一即可** |
| **Threshold** | 已是純黑白的圖（外部處理過）；非照片模式 |
| **Ordered/Bayer** | 大面積填色、復古網點風；人像少用 |
| **Newsprint/Halftone/Sketch** | 風格化用途，非還原度用途——長尾 |
| **Grayscale（變功率）** | **CO2 的 3D 浮雕深度雕刻**（圖要特製）；**二極體平面照片強烈不建議**（材質標記是二值的，灰階做不出中間調——「新手請用 dither」是共識） |

機型模式：二極體用戶幾乎都活在 dither 家族；Grayscale 是 CO2/浮雕工具。

## 比演算法更重要的硬數字（社群最常糾正的錯誤）

- **DPI 必須匹配實際光斑**：0.2mm 光斑 → 127 DPI；0.1mm → 254 DPI；0.05mm → 508 DPI。「對到焦的二極體超過 318 LPI 就開始疊線」。超過 → 相鄰線融成黑塊、細節變泥。
- **控制器陷阱**：8-bit GRBL 約 250 指令/秒；254 DPI × 50mm/s 產生約 500 指令/秒 → 掉點。很多「Jarvis 雕壞了」其實是 buffer 爆掉，跟演算法無關。
- **石板/深色石材要先反轉**（雕掉的地方變亮）；磁磚照片走 **Norton White Tile 法**（TiO₂ 塗層）——是材料前處理技術，不是演算法。
- **雙重抖動**：已抖動過的圖再進軟體會被重抖 → 需要 Pass-Through 模式。

## 給產品的結論

### (a) 只做 3–4 種就夠（覆蓋 ~90% 實際使用）

1. **Atkinson**（預設）2. **Jarvis**（大圖/平滑）3. **Stucki 或 Floyd 擇一**（細節型）4. **Threshold ＋ Pass-Through**（外部處理圖直出，防止重抖）。
Grayscale 保留但**加護欄**：標注「CO2 浮雕用」、二極體選它時主動勸退。Newsprint/Halftone/Sketch/Bayer/Sierra 可略或收進進階選單。

### (b) 比加演算法更有效的 UX（社群證據指向的真槓桿）

1. **DPI 自動匹配光斑**——我們知道每台機器的光斑規格，可直接算出建議 interval 並在超標時警告（修掉頭號新手錯誤）。
2. **材質配方預設**——以「材質」而非「演算法」思考：木頭/石板/磁磚/皮革/陽極鋁各自綁定「模式＋DPI＋是否反轉＋注意事項」的完整配方（與 EasySet 級材質庫方向合流）。
3. **前處理自動化**——對比/levels/銳化先於抖動；偵測已是 1-bit 的圖自動建議 Pass-Through。
4. **測試格按鈕化**——社群的「test, test, test」儀式；**我們已有材質測試產生器**，把它與照片模式打通即可。
5. 二極體＋照片＋選 Grayscale → 主動引導到 dither。

### (c) 幾句代表性的社群原話

- 「它們只是排列點的不同方法。」
- 「你的雷射能打出 0.1mm 的點嗎？你的電源在這個速度下來得及點出 0.1mm 的點嗎？」——把「選哪個演算法」重新框成硬體問題
- 「CO2 的灰階模式是給 3D 雕刻用的，很難調好。我們大多用 dither，我自己 Stucki 效果最好。」
- 「人像或 logo，從 Atkinson 開始。」
- 「最重要的事情是：測試。」

## 對差距分析的影響

第一級第 1 項「照片抖動套件」的實作範圍修正為：**3–4 種演算法＋DPI 自動匹配＋材質配方＋前處理**。競爭話術也隨之改變：不是「我們有幾種演算法」，而是「我們讓新手不用懂演算法就雕得好」——這比 LightBurn 的 8 種選單更符合 FLUX 的定位，也是 Studio 的 AI Assistant 正在走的方向。
