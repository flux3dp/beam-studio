# Note of image-edit-panel.spec.ts

## 對應測試表項目

右側物件面板分類：「圖像編輯 — 確認圖像編輯的功能是否可正常使用」。經確認此列指的是**獨立的「Edit Image」全視窗編輯器**（`packages/core/src/web/app/components/ImageEditPanel/`，Konva 畫布，自物件面板的 `#imageEditPanel` 按鈕開啟）——與 `image.spec.ts` 覆蓋的物件面板操作（漸層、取代影像、曲線、銳化、裁剪、斜角、反轉）是不同的功能面，無重疊。

## 測試了什麼

共 6 個測試（客戶端 Konva 運算，CI 可跑）：

1. **視窗開啟**：三個工具分頁（橡皮擦／魔術棒／圓角）存在、預設為橡皮擦、undo/redo 起始為停用、OK 按鈕存在；切換分頁會顯示對應控制項（筆刷大小／容差／半徑）。
2. **橡皮擦**：設定筆刷大小、於畫布上畫一筆，undo 變為可用；undo/redo 往返時按鈕狀態正確切換。
3. **魔術棒**：設定容差、點擊影像，操作寫入歷史（undo 可用）。
4. **圓角**：以輸入框設定半徑並寫入歷史；undo 還原為 0。
5. **套用（OK）**：編輯後按 OK，視窗關閉且畫布上 `#svg_1` 的 `xlink:href` 內容確實改變。
6. **取消（Back）**：編輯後從「Back to Beam Studio」離開，`xlink:href` 不變。

## 設計理由

- 「功能可正常使用」在此定義為：每個工具能啟動、能造成一次可觀察的操作（歷史紀錄／按鈕狀態），且套用/取消的最終效果正確（href 是否改變）——結構性斷言，不比像素。
- 兩個非顯而易見的實作點已記錄於 spec 註解：關閉斷言不能用「Edit Image」文字（物件面板按鈕同名）改驗 `.konvajs-content` 消失；橡皮擦的 mousedown/mouseup 之間需要一次 requestAnimationFrame 讓 React 狀態刷新（否則筆畫被靜默丟棄）——後者對真實使用者不構成問題（人類操作必跨 frame），但值得知道。

## 充分性分析

- 視窗內三個工具、undo/redo、套用/取消皆覆蓋，符合此列「逐一開啟驗證可正常使用」的意圖。
- 未覆蓋：編輯**結果的視覺正確性**（擦除範圍是否精準、魔術棒選區是否符合預期）——無像素比對工具，維持人工抽查；若未來建立視覺回歸機制可補。
