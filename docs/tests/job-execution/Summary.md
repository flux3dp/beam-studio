# 送工作 (job-execution)

## 本分類測試範圍

涵蓋送出工作後的執行控制與監看（測試表中「送工作」以兩個群組出現，此處合併）：停止功能、暫停與繼續功能、預估時間與實際工作時間是否接近，以及 Dashboard 上送出工作後的進度條與剩餘時間顯示是否正確。

本分類項目多屬機器行為或需連上實機，尚無專屬的自動化測試。

## 測試檔案清單

| 測試檔案 | 類型 | 涵蓋內容摘要 | 來源 |
| --- | --- | --- | --- |
| `apps/web/cypress/e2e/machine/dashboard-readonly.spec.ts` | Cypress E2E（實機唯讀） | 待機時開啟 Dashboard（Monitor 走 Mode.FILE）：標題含機器名稱＋即時狀態、File／Camera 分頁存在、Task 分頁不存在（守門未送工作）；不送任何工作。本地批次 `--machine-readonly` | Claude 自動產生 (2026-07) |

其餘項目多屬機器行為或需實際執行工作，維持人工。

## 尚未自動化項目

- 停止功能是否正常 — 屬機器行為，維持實機人工。
- 暫停與繼續功能是否正常 — 屬機器行為，維持實機人工。
- 確認預估時間與實際工作時間是否接近 — 預估時間的計算邏輯可以 Jest 單元測試；與實際工作時間的比對需**實際送工作執行**（Tier C），本輪唯讀範圍不涵蓋。
- 確認 Dashboard：**送出工作後**進度條與剩餘時間顯示是否正確 — 需實際送工作，維持人工；**待機時 Dashboard 可開啟並讀狀態**已由 `dashboard-readonly.spec.ts` 覆蓋。
