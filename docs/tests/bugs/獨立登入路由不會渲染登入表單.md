# Bug：獨立登入路由 `/#/initialize/connect/flux-id-login` 在 web 版不會渲染登入表單

- **狀態**：未解（已建立修復任務 task_6ebd684c）
- **嚴重性**：中 — 既有 Cypress 共用指令 `cy.loginAndLandingEditor` 因此完全失效；若使用者以此路由進入也會卡住
- **檔案**：路由/初始化流程（來源待查）；`apps/web/cypress/support/commands.ts` 第 117 行起的 `loginAndLandingEditor`

## 問題描述

`cy.loginAndLandingEditor` 造訪 `/#/initialize/connect/flux-id-login` 後等待 `input#email-input`，但該路由在 web 版（Cypress 環境）不會渲染出登入表單，導致逾時。實際可用的登入路徑是：進入編輯器 → Account 選單 →「Log in or Sign Up」→ 登入對話框正常出現。

另外，經由 Account 選單登入成功後**不會**出現「Successfully logged in. OK」確認視窗（`loginAndLandingEditor` 卻在等它），而是直接開啟 FLUX Credit 對話框。

## 重現條件

1. `pnpm nx run web:start` 啟動 dev server。
2. 瀏覽器開 `http://127.0.0.1:8080/#/initialize/connect/flux-id-login`。
3. 觀察頁面：登入表單（`#email-input`）未出現。
4. 或在 Cypress 中呼叫 `cy.loginAndLandingEditor()`，`#email-input` 等待逾時。

## 影響範圍

- 所有依賴 `loginAndLandingEditor` 的既有 spec（多為需實機的本地測試）在本地 rig 上會失敗。
- 新寫的 `top-bar/flux-id-login.spec.ts` 已改走 Account 選單流程（可作為修復指令時的參考實作，含 `testIsolation: false` 的 session 共享寫法）。

## 待釐清

- 是路由改版造成（initialize flow 條件式跳轉），還是該路由本來就只給特定情境（例如初次設定機器）使用？
- 修復方向：修路由，或改寫 `loginAndLandingEditor` 走 Account 選單流程。
