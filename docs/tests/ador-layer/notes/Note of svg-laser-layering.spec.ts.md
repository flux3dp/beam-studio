# Note of svg-laser-layering.spec.ts

## 對應測試表項目

Ador Layer 分類：「移動或切換圖層模組 — 匯入SVG選擇雷射頭，選擇依據圖層/顏色/不分層的pop-up，確認匯入後圖案正確」。與 `svg-pdf-ai.spec.ts` 互補：該 spec 覆蓋列印頭側的變體，本 spec 補齊雷射頭側的三種分層方式。

## 測試了什麼

共 3 個測試（需 FLUXGhost，CI 自動略過，本機以 `pnpm run cy:fluxghost` 執行；工作範圍 Ador、匯入 `svg.svg`、模組一律選雷射）：

1. **依圖層**：驗證雷射頭的分層 pop-up 三個選項齊全（圖層／顏色／不分層——與列印頭沒有「顏色」選項形成對照）；選「依圖層」後，來源 SVG 的圖層名稱被保留（圖層 1、預設圖層）、`use`/`symbol` 結構正確、且兩層皆為雷射模組（非列印、非全彩）。
2. **依顏色**：輕量驗證（深度斷言在 `svg-pdf-ai.spec.ts` 的顏色測試）——分成 `#3F51B5` 與 `#333333` 兩個以色碼命名的雷射圖層。
3. **不分層**：所有內容收進**單一**圖層（既有的預設 Layer 1）；幾何驗證「匯入後圖案正確」：兩個來源圖形合併為一個 `use`，物件面板尺寸約 30×20mm（容差 1mm），符合 svg.svg 內矩形＋橢圓的聯集範圍。

## 設計理由

- 三種分層方式各驗證其「結構性結果」（圖層數、命名、模組屬性、幾何尺寸），不用像素或平台相依的 checksum。
- 後端接線與 `svg-pdf-ai.spec.ts` 完全一致（`ghostPort` 環境變數注入、不寫死埠號），維持單一慣例。

## 充分性分析

- 此列的三個變體與 pop-up 選項完整覆蓋；搭配 `svg-pdf-ai.spec.ts`（列印頭側），SVG 匯入分層的 pop-up 矩陣兩側都有著落。
- 未覆蓋：PDF／AI 檔案的同流程（需完整版 FLUXGhost，見 svg-pdf-ai 筆記的相同限制）。
