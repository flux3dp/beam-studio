Beam Studio — Direct Paste from AutoCAD

Development prompt history & decision log

Project: fork of flux3dp/beam-studio  ·  Goal: copy in AutoCAD → paste vectors into Beam Studio (no DXF file save)  ·  Generated with Claude Code
User
Shared https://github.com/flux3dp/beam-studio (laser cutter workspace). Currently it requires saving DXF files from AutoCAD to import. Asked: can the API be modified to let users directly copy a design from AutoCAD and paste it into Beam Studio to print?
Assistant
Analyzed both sides of the problem.

How import works today

  • importDxf(file: Blob) reads text → parses with dxf2svg → SVG → canvas.
  • A global paste handler already exists (handles clipboard Files and HTML <img>).

Why direct vector paste fails

  • AutoCAD Ctrl+C puts a binary DWG (temp file) + EMF metafile on the clipboard — not DXF text.
  • The browser/Electron paste event only exposes text/plain, text/html, image/*, Files — never AutoCAD.r17 or CF_ENHMETAFILE.
  • Best case a naive paste recovers a raster image — useless for cutting.

Options proposed

  • A (recommended): AutoCAD-side helper puts DXF text on the clipboard; small Beam Studio change to import it.
  • B: Watched-folder / "Import latest" auto-import.
  • C: Accept raster paste only (loses vectors).
User
option A
Assistant implemented
Built both halves of Option A.

1. Beam Studio paste handler

Added a DXF-text branch to the global paste handler: detects DXF in text/plain, wraps it in a clipboard.dxf File, and reuses the existing handleFile → importDxf pipeline (unit dialog, undo, layers all preserved). Computed the DXF check up front so non-DXF text no longer pre-empts the image-paste branch.

2. AutoCAD BEAMCOPY command (AutoLISP)

Select objects → DXFOUT the selection to %TEMP%\beam_clip.dxf → pipe to clipboard with clip.exe.
